If creating visual artifacts (slides, mocks, throwaway prototypes, etc), copy assets out and create static HTML files for the user to view. If working on production code, you can copy assets and read the rules here to become an expert in designing with this brand.

If the user invokes this skill without any other guidance, ask them what they want to build or design, ask some questions, and act as an expert designer who outputs HTML artifacts _or_ production code, depending on the need.

## Quick map

- `README.md` — brand voice, visual foundations, iconography.
- `colors_and_type.css` — drop-in tokens (colours, type, spacing, radii, shadows).
- `fonts/` — licensed TTFs: League Spartan (display), Public Sans (body/web).
- `assets/` — logos, flower mark, brand PDF.
- `ui_kits/web/` — interactive homepage recreation + components.
- `preview/` — specimen cards for the Design System tab.

## The non-negotiables

- **Voice:** warm but not fluffy. Peer-to-peer, not broadcast. Specific over generic. No emoji. No AI-generated cadence.
- **Colour:** blue anchors, teal supports, orange highlights, purple sparingly. Never random.
- **Imagery:** real people, real contexts. Never circuit boards, robot hands, or blue brain graphics.
- **Four positioning guardrails:** systemic not reactive · realist not evangelist · human first · no political commentary.
